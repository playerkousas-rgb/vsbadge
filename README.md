# 🔥 深資童軍進度性獎章追蹤系統 v2.0

> 基於 2026 年第 11 版《深資童軍訓練綱要》  
> 支援四級進度追蹤、級聯自動完成、批量確認寫入

---

## 這是什麼？

一個追蹤深資童軍進度性獎章的系統，從會員章到榮譽童軍獎章，四級進度一目了然。

```
L1 會員章 → L2 肩章 → L3 深資童軍獎章 → L4 榮譽童軍獎章
```

- ✅ 領袖可以勾選完成項目，自動記錄日期
- ✅ 勾上層自動完成所有下層（級聯）
- ✅ 全團總覽 + 按項目交叉查看
- ✅ 成員可以看到自己的進度
- ✅ 瀏覽器自動備份（防當機）

---

## 如何使用？

本系統支援兩種模式，選一種適合你的：

### 模式 A：獨立使用（不需要主系統）

直接打開系統網址，在設定頁配置：

```
https://vs-badge-tracker.vercel.app/
```

首次開啟會看到設定畫面，填入：
- **旅團編號**（例：0082）
- **身份**（領袖 / 成員）
- **Apps Script 後端 URL**（見下方「部署後端」）
- **API Key**（你設定的）

設定會存入瀏覽器，下次自動帶入。建議加入書籤。

---

### 模式 B：接入主系統

如果你的旅團有使用 scoutsystem-2.0 主系統，可以把本系統接入：

1. 部署好後端（見下方）
2. 把 **Apps Script URL** + **API Key** 交給系統管理員
3. 管理員會在主系統元件設定中加入
4. 之後從 Dashboard 的元件卡片進入，身份自動帶入

**你需要交給系統管理員的資訊：**

| 資訊 | 說明 |
|------|------|
| 旅團編號 | 例：0082 |
| Apps Script URL | 你部署的後端網址 |
| API Key | 你設定的密鑰 |

就這些。其他的系統管理員會處理。

## 部署後端（Google Sheet）

**無論獨立使用或接入主系統，都需要先部署後端。**

### 步驟 1：建立 Google Sheet

1. 打開 [Google Sheets](https://sheets.google.com)
2. 建立新試算表，取名為「深資童軍進度追蹤」
3. 複製網址中的 Sheet ID：
   ```
   https://docs.google.com/spreadsheets/d/【這段就是ID】/edit
   ```

### 步驟 2：貼上後端程式碼

1. 在 Sheet 中點擊 **擴充功能** → **Apps Script**
2. 刪除所有現有內容
3. 複製下方 `Code.gs` 程式碼（聯絡系統管理者取得）
4. 全部貼上
5. 修改最上面兩行：

```javascript
const SHEET_ID = '你的Sheet ID';
const API_KEY = '你自己設定的Key';  // 例：'troop0082_vs2026'
```

6. 點擊 💾 **儲存**
7. 在函式選單選擇 `initializeSheets`，點擊 ▶ **執行**
8. 首次會要求授權 → 選擇帳號 → 進階 → 前往 → 允許

### 步驟 3：部署

1. 點擊 **部署** → **新增部署**
2. 點擊 ⚙ 齒輪 → 選擇 **網頁應用程式**
3. 設定：
   - 執行身分：我
   - 存取權：任何人
4. 點擊 **部署**
5. 複製產生的網址（以 `/macros/s/.../exec` 結尾）

### ✅ 完成！

現在你有：
- **Apps Script URL** → 用於設定
- **API Key** → 你剛才設定的

把這兩個交給系統管理員（模式B），或自己填入設定頁（模式A）。

```
4 個段章全 ✓     → 深資童軍獎章自動完成
```

## 常見問題

**Q: 我該用獨立使用還是接入主系統？**  
A: 如果你的旅團有使用 scoutsystem-2.0，建議接入主系統（身份自動帶入更方便）。如果沒有，用獨立使用就好。

**Q: 兩種模式可以同時用嗎？**  
A: 可以。領袖從主系統進入，成員直接 bookmark 前端 URL。

**Q: 需要密碼嗎？**  
A: 獨立使用不需要密碼，靠 API Key 保護。接入主系統時身份由主系統帶入。

**Q: 資料存在哪裡？**  
A: 存在你自己建立的 Google Sheet 裡。你完全擁有資料。

**Q: 忘記 API Key 怎麼辦？**  
A: 打開 Apps Script 編輯器，查看 `Code.gs` 最上面的 `API_KEY`。

**Q: 勾選後沒有保存？**  
A: 記得按底部的「確認寫入 Google Sheet」按鈕。勾選時資料暫存在瀏覽器，只有按確認才會寫入。

## 技術細節

### URL 參數

| 參數 | 說明 |
|------|------|
| `u` | 旅團編號 |
| `role` | `member` / `leader` 等 |
| `ymis` | 成員 YMIS |
| `embed` | `1` = 嵌入模式（隱藏頂部） |
| `backend` | Apps Script URL |
| `apikey` | API Key |

### Google Sheet 結構

| 工作表 | 內容 |
|--------|------|
| 成員名單 | ymis, name |
| 進度紀錄 | ymis, item_id, item_name, completed_date, updated_by |
| 項目定義 | item_id, display_name, badge, segment（靜態） |

### 安全性

- ✅ API Key 鎖定：沒有 Key 無法存取
- ✅ HTTPS：必須
- ✅ 資料隔離：各旅團獨立 Sheet
- ✅ 身份驗證：主系統帶入或設定頁帶入
