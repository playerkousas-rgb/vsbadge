# 深資童軍進度APP v4.0 修復報告 - 回應 11 項需求

## 1. LOGO 顯示不到 ✅ 已修復
**問題原因：**
- 舊版使用 `<img src="assets/vs-logo-small.png">`，該 PNG 1.3M 過大，Vercel 上相對路徑在嵌入模式下失效，且沒有 fallback。
- 出現 404 或載入過慢時頁面顯示空白。

**修復方案：**
- 改用內嵌 SVG 火炬標誌（原 `assets/vs-torch.svg` 優化版），以 `<div class="logo-svg-wrap"><svg>...</svg></div>` 永久嵌入 HTML，無需外部請求，任何環境皆顯示。
- 保留 PNG 嘗試載入作備用，但主視覺完全不依賴外部文件。
- Header 加入漸變、毛玻璃效果，視覺更佳。

## 2. 百分比那頁卡著了 ✅ 已修復
**問題原因：**
- `index.html` 中 nav 按鈕與 tab 內容共用同一 ID：`overviewTab`, `byitemTab`, `usersTab`, `appsTab` 各出現兩次，`document.getElementById` 取得錯誤元素，`switchTab` 時死 loop。
- 全團總覽頁一次渲染所有項目（100+欄）× 成員數，DOM 瞬間產生 5000+ 節點，低階手機直接卡死。
- 按項目統計頁無分頁，計算百分比時重複查詢 `progressData[m.ymis]` 但舊結構是 `progressData[ymis][itemId]` 扁平對象，導致 `undefined` 錯誤。

**修復方案：**
- 重命名 ID：導航按鈕改為 `btn-tab-*`，內容改為 `tab-*`，徹底分離。
- `switchTab` 改用 `event.currentTarget` 而非 `event.target`，避免點擊 span 時失效。
- 全團總覽改為：
  - 頂部篩選器：選擇獎章、段章、搜尋成員
  - 限制最多 30 欄，若超過顯示警告並提供「仍要顯示全部」按鈕
  - 成員限制50人內顯示，卡片式總覽 + 虛擬表格，避免巨型表格
- 按項目頁改為搜尋 + 獎章篩選，逐項計算，不再一次渲染全部。
